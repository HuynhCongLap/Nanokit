Présentation de NanoEdit

NanoEdit fonctionne uniquement sous linux.

Sur les machines du bâtiment Nautibus

Vous pouvez utiliser NanoEdit sur n'importe quelle machine du bâtiment.
Pour cela il suffit de créer un répertoire sur votre répertoire de connexion et d'y copier ce makefile .
Pour lancer NanoEdit, il suffit de lancer la commande make
NB : Vous n'avez pas besoin d'installer les sources de NanoEdit sur votre compte à l'Université.

Installation sur votre machine personnelle

Pour l'installation, consultez cette page.

Après l'installation, vous trouverez 2 répertoires sur votre répertoire de connexion : NanoEdit et MonNano.

NanoEdit contient les sources de NanoEdit que vous aurez compilés en suivant les indications de la page d'installation.

Le répertoire MonNano contient un fichier makefile. Pour lancer NanoEdit, il suffit de lancer la commande make
Vous mettrez dans ce répertoire vos fichiers sources (cf. le paragraphe "Création d'objets").

Utilisation de NanoEdit

  • Allez dans votre répertoire MonNano
  • Tapez la commande make pour lancer NanoEdit
Pour pouvez créer des objets en cliquant dans le menu création ou bien en utilisant les raccourcis claviers.
Voici les raccourcis que vous utiliserez le plus souvent :
  • P3 : création d'un point 3D
  • P4 : création d'un point 4D
  • TP3 : création d'une table de points 3D à partir d'une liste de P3 sélectionnés
  • TP4 : création d'une table de points 4D à partir d'une liste de P4 sélectionnés
  • GP3 : création d'une grille de points 3D à partir de listes de TP3 sélectionnées
  • GP4 : création d'une grille de points 4D à partir de listes de TP4 sélectionnées
  • co : création d'un objet couleur
  • Touche F1 sur un objet sélectionné : affichage de la fenêtre d'édition de cet objet
  • Touche F3 : désélection de tous les objets sélectionnés
sauvegarde de scènes, chargement de fichiers, compilation à partir de l'application ...

Création d'objets

Code écrit en langage C (C++ possible)
  • Les fichiers sources dont le nom commence par o_ sont les fichiers contenant les "objets" de NanoEdit.
  • De préférence un objet par fichier, c'est plus simple et plus lisible
  • Pour compiler il suffit de lancer la commande make, vous n'avez pas à modifier le fichier Makefile
Fichiers d'exemples pour la création d'objets dans NE :
  • figure.c
  • figure.h
  • o_cerc.c
  • o_sph.c
  • o_type_mg.c.memo fichier regroupant les types les plus fréquents et leur déclaration dans une CLASSE

À vous de jouer

  1. Sauvegarder les exemples précédents sur votre compte, lancez NanoEdit et testez-les
  2. Testez les différentes fonctionnalitées de NanoEdit montrées pendant la démo.
  3. Écrivez un objet NanoEdit qui crée un carré centré sur l'origine du repère (0,x,y) et de dimension 1.
    Les 2 paramètres centre du carré et dimension du carré seront des paramètres éditables . Testez votre objet
  4. Modifiez votre code pour que ces 2 paramètres soient extraits de NanoEdit. Testez-le
  5. Modifiez votre code pour que ces 2 paramètres soient extraits de NanoEdit et obligatoires. Testez-le
    (la deuxième version est préférable pour l'utilisateur)
  6. Ajoutez un champs Booleen permettant de basculer d'un affichage filaire à un affichage "plein" (carré colorié)
    7. Ajoutez un champs Menu permettant de sélectionner une couleur pour l'affichage du carré parmi un choix de 5 couleurs
  7. Sauvegardez l'objet que vous avez créé dans un fichier
  8. Quittez NanoEdit, relancez NanoEdit et chargez le fichier que vous avez créé précédemment
Si vous avez terminez avant la fin de la séance :
  1. Écrivez un objet NanoEdit qui affiche une liste de points P3 à l'aide d'une ligne brisée.
  2. Ajoutez un champs Booleen permettant de basculer de l'affichage du polygone à l'affichage du polygone fermé. La liste des points sera stockée dans une table de points P3 Extraite
